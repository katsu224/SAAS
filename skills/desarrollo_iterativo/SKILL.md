# Skill: Desarrollo Iterativo y Modular

Esta skill impone un flujo de trabajo riguroso y comunicativo para asegurar que el desarrollo se alinee perfectamente con las expectativas del usuario, especialmente en proyectos complejos.

## Flujo de Trabajo Obligatorio

### 1. Fase de Clarificación (ANTES del Plan)

**Objetivo**: Entender completamente el "qué" y el "por qué" antes del "cómo".

- **Acción**: Haz una pausa antes de generar cualquier código o plan técnico.
- **Preguntas**: Formula preguntas claras y directas en español para despejar dudas sobre lógica de negocio, preferencias de diseño o alcance.
- **Formato**:
  > "Para asegurar que estamos alineados, necesito que me confirmes lo siguiente:"
  >
  > 1. [Pregunta sobre lógica A]
  > 2. [Pregunta sobre diseño B]
  >    "¿Estás de acuerdo con este enfoque?"

### 2. Fase de Definición de Módulos (Proyectos Grandes)

**Objetivo**: Dividir el problema en partes manejables.

- **Acción**: Si el proyecto es complejo, propón una división por módulos o secciones.
- **Ejemplo**:
  - Módulo 1: Página de Inicio (Landing).
  - Módulo 2: Autenticación.
  - Módulo 3: Panel de Administración.
- **Validación**: Pide confirmación al usuario sobre este orden de ejecución.

### 3. Ejecución Iterativa (Paso a Paso)

**Objetivo**: Calidad sobre velocidad. Evitar re-trabajo masivo.

- **Regla de Oro**: **NO** procedas al siguiente módulo hasta que el usuario haya aprobado explícitamente el actual.
- **Proceso por Módulo**:
  1.  **Implementar**: Escribe el código del módulo actual.
  2.  **Verificar**: Asegura que compila/funciona.
  3.  **Presentar**: Muestra el resultado al usuario (capturas, archivos, logs).
  4.  **Preguntar**: "¿Podemos dar por finalizado este módulo y pasar al siguiente?"

## Estilo de Comunicación

- **Idioma**: Español.
- **Tono**: Colaborativo pero estructurado.
- **Validación Continua**: Nunca asumas que el silencio es aprobación. Busca un "sí" explícito.

## Cuándo usar esta Skill

- Cuando el usuario dice "hazme preguntas primero".
- En proyectos grandes desde cero.
- Cuando el usuario pide "ir paso a paso".
